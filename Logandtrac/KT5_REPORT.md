# КТ 5. Логирование ошибок и исключений

## Что реализовано

1. Добавлена централизованная обработка исключений:
- в `TaskManagerService` через метод `ExecuteOperationSafely(...)`;
- в `Program` через метод `ExecuteCommandSafely(...)`.

2. Для всех перехваченных исключений введен единый формат логирования:
- `Exception.Message`;
- `Exception.StackTrace`;
- контекст операции (`operation` + `context`);
- уровень критичности (`Error`, `Fatal` и др.).

3. Добавлен общий API логирования исключений:
- `StructuredLogger.LogException(...)` в `Logging/StructuredLogger.cs`.

4. Реализовано опциональное оповещение:
- при исключениях выводится консольное уведомление:
  `!!! Ошибка !!! Проверьте логи для деталей.`

## Где внесены изменения

- `Logging/StructuredLogger.cs`
  - добавлен метод `LogException(...)` для централизованной записи исключений.
- `Services/TaskManagerService.cs`
  - добавлен метод `ExecuteOperationSafely(...)`;
  - операции `AddTask`, `RemoveTask`, `ListTasks` обернуты в централизованный обработчик.
- `Program.cs`
  - добавлен метод `ExecuteCommandSafely(...)`;
  - диспетчер команд выполняется через безопасный вызов;
  - фатальные ошибки `Main` пишутся централизованно с уровнем `Fatal`.

## Результат

- Исключения обрабатываются централизованно.
- Логи содержат полный контекст для диагностики:
  сообщение, стек, операция, уровень.
- Пользователь получает мгновенное визуальное предупреждение в консоли.
- Система готова к расширению под внешние уведомления (Sentry/Application Insights/email) через `StructuredLogger.LogException(...)`.
